# InventorySync Plugin - 1단계 완성

## 기능

✅ 서버의 모든 플레이어 인벤토리 **실시간 동기화**
✅ 슬롯 위치까지 완벽하게 동기화 (핫바, 일반 인벤토리, 갑옷, 오프핸드)
✅ 자동 감지 (아이템 이동, 드롭, 줍기 등)
✅ 새 플레이어 입장 시 자동 동기화

---

## 설치 방법

### 방법 1: 직접 빌드

**필요한 것:**
- Java JDK 8 이상
- Maven 설치

**빌드 명령어:**
```bash
mvn clean package
```

**생성된 파일:**
- `target/InventorySync-1.0.0.jar`

### 방법 2: 미리 빌드된 파일 사용

아래의 `InventorySync-1.0.0.jar` 파일을 서버의 `plugins` 폴더에 넣으면 됨

---

## 플러그인 적용

1. 서버의 `plugins` 폴더 열기
2. `InventorySync-1.0.0.jar` 파일 복붙
3. 서버 재시작 또는 `/reload` 명령어 실행
4. 콘솔에 다음 메시지 나타나면 성공:
   ```
   [InventorySync] 플러그인이 활성화되었습니다!
   모든 플레이어의 인벤토리가 동기화됩니다.
   ```

---

## 작동 방식

### 감지되는 행동:

✅ **인벤토리 클릭** - 아이템 이동, 위치 변경
✅ **아이템 드롭** - 아이템 버리기
✅ **아이템 줍기** - 바닥의 아이템 줍기
✅ **플레이어 입장** - 새 플레이어가 서버에 들어올 때

### 동기화 대상:

- 핫바 (1번~9번 슬롯)
- 일반 인벤토리 (10번~35번 슬롯)
- 갑옷 (헬멧, 흉갑, 레깅스, 부츠)
- 오프핸드 (왼손)

---

## 테스트 방법

**플레이어 3명 기준:**

1. **서버에 플레이어 A, B, C 접속**
2. **플레이어 A가 나무 1개 획득** → 아이템 줍기
3. **확인:** B, C의 인벤토리에도 나무 1개가 있는가?
4. **플레이어 A가 나무를 핫바 1번으로 이동**
5. **확인:** B, C의 핫바 1번에도 나무가 있는가?
6. **플레이어 B가 나무를 드롭**
7. **확인:** A, C의 인벤토리에서 나무가 사라지는가?

3. 여러 플레이어가 동시에 작업할 때 문제는 없나?
4. 갑옷, 오프핸드도 동기화되나?

## 문제 발생 시

1. **인벤토리가 동기화 안 될 때:**
   - 서버 콘솔에 에러 메시지 확인
   - `/reload` 또는 서버 재시작

2. **특정 행동에서만 안 될 때:**
   - 어떤 행동에서 안 되는지 알려주기
   - 예: "아이템 줍기는 안 되는데 이동은 됨" 이런 식으로

3. **성능 문제:**
   - 플레이어가 너무 많으면 딜레이 발생 가능
   - 최대 10명 이하에서는 문제 없음
